### 1-3. ホバーツールチップの表示崩れと修正（L100〜L160）

**geminiの参入（L104〜）**:
gemini がホバーツールチップをANSIカラー付きで実装。しかしすぐに「表示崩れ」が発生。

**表示崩れの原因調査（長期にわたるデバッグセッション）**:

問題の症状:
- ツールチップが画面からはみ出す
- テキストが横に広がりすぎる
- `white-space: pre-wrap` にしたら改行は正しいが横幅が膨大になる

試した修正（失敗したもの）:
1. `max-width: 600px` → テキストがはみ出る（terminal padding による末尾スペース）
2. `overflow: hidden` → 情報が切れる
3. `white-space: pre` → 横スクロールバーが出る
4. CSS の `word-break: break-all` → 単語が途中で切れて読みにくい

**根本原因の特定**: tmux の `capture-pane` コマンドは、各行をターミナルの幅（例: 220文字）まで末尾スペースで埋める。つまり実際の内容が20文字しかなくても200文字のスペースが追加される。

**正しい修正（gemini）**: Python側の `/trace` エンドポイントで各行に `rstrip()` を適用。これにより末尾スペースが削除され、表示崩れが解消された。これはコミット `b1386fb` の一部として記録された。

追加で `dynamic max-width`（内容に応じて幅が変わる）も実装。

### 3-1. モバイルコンポーザーの継続的修正（L1941〜L2100）

**グリッド vs フレックスレイアウト**:
`+` ボタンとテキスト入力と送信ボタンの並び方について、gridとflexの使い分けで意見が分かれた。


### 4-1. リプライプレビューと@ファイルドロップダウンの干渉（L3472〜L3530）

**問題**: メッセージへの返信中（リプライバナーが表示中）に `@` を打ってファイルドロップダウンを開こうとすると、ドロップダウンをタップした瞬間にリプライバナーが消えてしまう。

**原因分析（gemini）**:
`blur` イベントの発火順序の問題。
1. textarea がフォーカスを持っている
2. ユーザーがドロップダウン内のアイテムをタップ
3. `blur` イベントが先に発火 → リプライバナーを閉じる処理が走る
4. ドロップダウンの `click` イベントが発火

**修正（gemini）**:
`blur` イベントハンドラに `setTimeout(fn, 150)` のディレイを追加。クリックイベントが先に処理されてから blur ハンドラが実行されるようにした。

### 4-2. ターゲットチップとキーボードの問題（L3600〜L3700）

**問題**: モバイルで、返信先エージェントを選択するターゲットチップ（「claude」「codex」など）をタップすると、キーボードが出た状態では画面がジャンプする。また、キーボードが開いているときにチップをタップするとキーボードが閉じてリプライプレビューも消える。

**修正**: チップのタップ時に `e.preventDefault()` を追加してキーボードの自動管理を防止。チップのフォーカス処理を明示的に制御。

## 第5章: 3月12日（1257メッセージ、L4207〜L5463）— フォント導入とモバイルセッションハブ

### 5-1. Anthropic Serifフォントのインストール（L4207〜L4400）

**codexによる調査と実装**:

Anthropicのウェブサイトで使われているフォント名を特定: `"Anthropic Serif"` (カスタムフォント、`Claude.app` バンドルに内包)

インストール方法:
```bash
# Claude.app バンドルからフォントファイルを取得
find /Applications/Claude.app -name "*.otf" -o -name "*.ttf"
# → Anthropic-Serif-Regular.otf, Anthropic-Serif-Italic.otf 等を発見
# Fonts フォルダにコピー
cp /Applications/Claude.app/.../Anthropic-Serif-*.otf ~/Library/Fonts/
```

CSS での使用:
```css
@font-face {
  font-family: "Anthropic Serif";
  src: url("file:///.../Anthropic-Serif-Regular.otf");
}
```

**問題1: CSSの特異性（specificity）問題**:

`font` ショートハンドプロパティが個別の `font-family` を上書きしてしまう。

例:
```css
.message {
  font: 14px/1.5 sans-serif;  /* これが後から適用されてserifを上書き */
  font-family: "Anthropic Serif";  /* これが効かない */
}
```

原因: `font` ショートハンドは `font-family`, `font-size`, `line-height`, `font-weight` を一括設定する。後から個別の `font-family` を指定しても、CSSのカスケード順で同じ specificity なら後の宣言が優先されるが、コードの中で `font` ショートハンドが後に書かれていた。

**修正（codex）**:
`font` ショートハンドをすべて個別プロパティに分解:
```css
.message {
  font-family: "Anthropic Serif", Georgia, serif;
  font-size: 14px;
  line-height: 1.5;
  font-weight: 400;
}
```

**問題2: フォントのパス問題**:
`file://` URL はセキュリティ制限でブラウザから読めない（Same-origin policy）。

**修正**: `agent-index` サーバーの `/font/<name>` エンドポイント経由でフォントファイルを配信。
```css
@font-face {
  font-family: "Anthropic Serif";
```

### 6-1. ハブのテストと改良（L5464〜L5600）

**問題1: オーファンサーバー**

Killしたセッションのチャットサーバー（Python HTTPサーバー）がプロセスとして残り続ける。

症状:

問題の詳細:
1. ユーザーが「送信」ボタンを押す
2. iOS Safari は先に `blur` イベントを textarea に発火させる

**修正**:
```javascript
sendButton.addEventListener('touchend', (e) => {
  e.preventDefault();  // デフォルトの blur をキャンセル
  submitMessage();      // 明示的に送信
  textarea.blur();      // 手動でキーボードを閉じる
});
```

### 6-6. `forceScroll` ヌル参照エラーの修正（L5925〜L5960）

## 第9章: 実装の技術的詳細

### 9-1. /trace エンドポイント

```python
@app.route('/trace')
def trace():
    agent = request.args.get('agent')
    pane = AGENT_PANES.get(agent)
    if not pane:
        return jsonify({'lines': []})

    result = subprocess.run(
        ['tmux', 'capture-pane', '-p', '-t', pane, '-e'],
        capture_output=True, text=True
    )
    lines = result.stdout.split('\n')
    # 末尾スペースを除去（terminal paddingの除去）
    lines = [line.rstrip() for line in lines]
    # 最新N行を返す
    lines = lines[-50:]
    return jsonify({'lines': lines})
```

### 9-2. @ファイルオートコンプリート（JavaScript概要）

```javascript
textarea.addEventListener('input', async () => {
    const val = textarea.value;
    const atPos = val.lastIndexOf('@');
    if (atPos === -1) { closeDropdown(); return; }

    const query = val.slice(atPos + 1);
    const res = await fetch(`/files?q=${encodeURIComponent(query)}`);
    const { files } = await res.json();

    showDropdown(files);
});

// Enter/Tab で選択
textarea.addEventListener('keydown', (e) => {
    if (dropdown.visible && (e.key === 'Enter' || e.key === 'Tab')) {
        e.stopImmediatePropagation();  // 送信を防ぐ
        insertFile(selectedFile);
    }
});
```

### 9-3. モバイルセッションハブのデータフロー

```
iPhone → GET /hub
         ↓
      tmux list-sessions
         ↓
      各セッションのメタデータ読み込み (logs/<session>/.meta.json)
         ↓

         ↓
iPhone ← カードHTML

iPhone → POST /hub/kill/<session>

         ↓
iPhone ← { "status": "killed" }
```

### 9-4. オーファンサーバー問題の修正コード

```bash
# セッション起動時（multiagent スクリプト内）
echo $! > "logs/${SESSION}/.server.pid"

# Kill時（hub の /kill エンドポイント）
PID_FILE="logs/${session}/.server.pid"
if [ -f "$PID_FILE" ]; then
    kill $(cat "$PID_FILE") 2>/dev/null
    rm "$PID_FILE"
fi
tmux kill-session -t "$session" 2>/dev/null
```
